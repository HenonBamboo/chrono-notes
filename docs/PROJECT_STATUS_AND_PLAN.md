# StickyNotesC 项目说明与修改计划

更新日期：2026-05-24

## 1. 项目定位

StickyNotesC 是一个 Windows 桌面便签应用，目标不是做成 Notion 或完整项目管理系统，而是做一个轻量、好看、输入快、能按时间阶段管理事件、能调用 AI 摘要的桌面便签。

当前主线已经确定为：

- UI：Qt 6 Quick/QML。
- 应用层：C++17，负责 Qt 模型、窗口状态、交互状态、数据持久化和异步 AI 调用。
- 数据层：C++17，便签持久化走 QtSql SQLite，配置和 AI HTTP 客户端也已转为 C++ 编译单元。
- 构建方式：CLion + CMake。
- 运行目标：`StickyNotesC`。

旧版 Win32、Direct2D、Nuklear、本地 Web UI 方案已从工作区移除，当前开发主线只保留 Qt/QML。

## 2. 当前核心功能

- 按每天、每周、每月、每年管理便签。
- 每条事件可以完成、取消完成、删除、编辑。
- 完成事件会记录完成时间，并排在未完成事件之后。
- 周、月、年视图自动收纳每日事件，收纳内容以只读方式展示。
- 新增事件自动进入当前阶段顶部。
- 支持全选完成，再次点击可取消全选。
- 支持删除、完成、取消完成后的 5 秒撤销。
- 支持 `Ctrl+F` 全局搜索全部阶段内容。
- 支持搜索关键词高亮和按完成状态筛选。
- 支持长内容右侧详情抽屉查看和编辑。
- 支持重复任务，完成日任务后按每天、每周、每月、每年生成下一条。
- 支持通过系统文件选择窗口导出 JSON、导入 JSON、导出 Markdown。
- 支持本地操作日志，记录新增、更新、完成、删除、撤销、清理、导入导出。
- 支持 AI 摘要历史保存。
- 支持 AI 摘要抽屉，采用 OpenAI-compatible API 配置方案。
- 支持设置抽屉，保存 API URL、API Key、模型名，并提供基础数据清理、导入导出。
- 支持无边框窗口、四边和四角拖拽缩放。

## 3. 已完成修改

### 3.1 工程主线清理

- 明确当前主线为 Qt/QML。
- 旧 UI 方案和废构建目录已移除，减少判断噪音。
- `CMakeLists.txt` 仅保留当前主线构建目标和测试目标。
- `README.md` 已改写为当前路线说明。

### 3.2 UI 与交互

- 主界面改为 Qt/QML 自绘无边框窗口。
- 加入四边和四角拖拽缩放。
- 抽屉式 AI 摘要与设置面板已拆分为独立 QML 组件。
- 列表项支持悬浮操作，完成、删除、详情不再常驻占位。
- 新增折叠式搜索条，默认不占空间。
- 新增详情抽屉，长内容不再强行塞在列表行里。

### 3.3 数据与状态逻辑

- 完成/删除后减少整页刷新，尽量使用模型移动、删除、布局变化。
- 删除和完成状态变化支持撤销。
- 测试运行使用临时数据目录，避免污染真实运行数据。
- 全局搜索不复制事件，只通过原始 `eventId` 操作原事件。
- 详情抽屉由 C++ 模型维护选中事件状态，避免 QML 自己猜数据状态。

### 3.4 AI 摘要

- AI 摘要改为后台异步调用，避免阻塞 UI。
- 设置中保留常规 OpenAI-compatible 接口方案。
- AI 摘要、设置、详情统一走右侧抽屉交互。

### 3.5 测试

- `note_store_tests` 覆盖 C 数据层基础能力。
- `config_tests` 覆盖配置读写。
- `qt_note_app_tests` 覆盖 Qt 应用层行为，包括：
  - 临时数据目录隔离。
  - AI 空要求处理。
  - 完成事件移动而非整页重置。
  - 删除撤销。
  - 完成/取消完成撤销。
  - 跨阶段搜索。
  - 清空搜索后恢复当前阶段。
  - 详情选中、更新、删除后的状态同步。
  - 旧文本存储迁移到 SQLite。
  - JSON 导出/导入、Markdown 导出。
  - 重复任务生成下一条。
  - 搜索完成状态筛选与关键词高亮。
  - 全局已完成清理。
  - 动态数据容量超过旧 512 上限。
  - 操作日志落盘。
  - AI 摘要历史落盘。
  - 自动收纳历史密集时默认折叠。

## 4. 本轮完成内容

本轮把计划里剩余的硬骨头直接收掉：

1. 主线全部切到 C++：
   - `src/` 和 `tests/` 下活动 `.c` 文件已消失。
   - `CMakeLists.txt` 改为 C++ 项目，活动源文件使用 `.cpp`。
   - C 语言层没有保留收益，继续混着只会增加 ABI、编码和构建复杂度。

2. 数据层升级：
   - 新增 SQLite 主存储 `notes.sqlite`。
   - 首次启动检测到旧 `notes.db.txt` 时自动迁移。
   - 文本格式读取继续兼容旧数据和带 repeat 字段的新文本格式。

3. 效率功能补齐：
   - 重复任务支持每天、每周、每月、每年。
   - JSON 导出/导入、Markdown 导出接入设置抽屉，并通过系统文件选择窗口选择路径。
   - 搜索支持完成状态筛选和关键词高亮。
   - 支持全局清理已完成事件。

4. 本地可靠性补齐：
   - `NoteStore` 从固定数组改为指针稳定的 `std::deque`，取消旧 512 条硬上限。
   - 用户动作写入 `operations.jsonl`，先把审计根基打上。
   - AI 摘要写入 `summary-history.md`，可回看要求、结果和输入快照。
   - 自动收纳历史超过阈值时默认折叠，手动点击统一的自动收纳组可展开。
   - 新增 `tools/package_release.ps1`，构建后生成包含 Qt 运行时的发布 ZIP。

## 6. 尚未完成的问题

- UI 视觉已经能跑，但距离“非常高级”的设计仍有差距，尤其是复杂数据状态下的密度控制。
- AI 接口只能保留通用配置，无法内置可公开分发的免费 Key。
- 没有云同步、账号系统、多设备同步；这些目前不建议做，先把本地体验做扎实。
- 当前提供 ZIP 发布包流程；如果后续要双击安装向导，再补 NSIS/WiX 安装器。
